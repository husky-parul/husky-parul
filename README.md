### Hi, I'm Parul Singh

Principal Software Engineer at Red Hat, Office of the CTO.

I work at the intersection of **AI agent security**, **cloud-native infrastructure**, and **sustainability**. My focus is on trust, identity, and observability for autonomous AI agents — building systems where machines can prove who they are, what they're allowed to do, and how much energy they burn doing it.

**SIG Lead, [CoSAI](https://www.coalitionforsecureai.org/) ADLC** — leading the Agent Development Lifecycle security workstream within the Coalition for Secure AI (an OASIS Open project with members including Google, Microsoft, OpenAI, Anthropic, and others). ADLC extends traditional SDLC frameworks to address the unique challenges of autonomous decision-making, non-deterministic behavior, and runtime tool invocation in AI agent systems.

#### What I'm Building

- **[trust-graph](https://github.com/husky-parul/trust-graph)** — Trust-aware dataplane for agentic systems — consistent authorization enforcement across multi-hop agent → tool → resource chains. Proposed in both [CoSAI WS4](https://github.com/cosai-oasis/ws4-secure-design-agentic-systems/issues/50) and [kagenti](https://github.com/kagenti/kagenti/issues/922)
- **[Kepler](https://github.com/sustainable-computing-io/kepler)** — Kubernetes energy observability using eBPF (CNCF Sandbox)
- **[PEAKS](https://github.com/husky-parul/peaks)** — ML-driven power-efficiency-aware Kubernetes scheduler
- **[modelcard-tools](https://github.com/husky-parul/modelcard-tools)** — OCI-native model card tooling for AI transparency
